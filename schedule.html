Week 1: The Internet and the World-Wide Web
Lecture 1 History and Architecture of the Internet. TCP/IP Review. Learning outcome: Refresh knowledge on Internet architecture, protocols and evolution. Gain an understanding of the principles behind Internet's successful design and evolution. Gain a good working knowledge of the basic building blocks of and the main keywords related to the Internet infrastructure. Readings:
  • Chapter 1. "Web Programming", Step, Miller & Kirst.
  • A video on the history of Arpanet, Internet's precursor: Computer Networks - The Heralds Of Resource Sharing (Arpanet, 1972)
Lecture 2 History and Architecture of the World-Wide Web. Learning outcome:
  • Get acquainted with the Definition, History, Evolution, and Current Challenges of the World-Wide Web. Understand the main elements of the World-Wide Web.
  • "As we may think," by Vannevar Bush, The Antlantic (July 1945).
  • The Web as a Distributed System. Overview of the HTTP Protocol. Components and Interactions of Web Applications.
 Readings:
  • Chapter 1. "Web Programming", Step, Miller & Kirst.
  • Internet Trends Report 2016 by Mary Meeker of Kleiner, Perkins, Caufield, Byers (Slides, ).
Precept 1 Introduction to the basics of the Web Infrastructure.
Learning outcome:
  • Learn the principles of the Client-Server computing model.
  • Learn the architecture and main components of Web application infrastructures.
  • Understand the key notions of Cloud Computing.
Readings:
  • The Client-Server Model (Wikipedia).
  • "Lessons from Giant-Scale Services" by Eric Brewer, IEEE Internet Computing (July/August 2001) (available in pdf from author's web site).
Lab 1 Introduction to Cloud Computing and Amazon Elastic Compute Cloud.
Learning outcome:
  • Understand the notion of virtual Linux servers.
  • Obtain hands-on familiarization with Virtualization concepts (physical vs virtual host, image vs instance) Brief overview of the Virtualization / Cloud Provider landscape (IaaS vs PaaS vs SaaS)
  • Introduction and focus on Amazon Web Services (AWS)
  • Registering a GitHub Student Developer pack and getting AWS free-usage credits
  • Create an Amazon Web Services (AWS) account.
  • Launch your own virtual server instance on Amazon.
  • Learn about Amazon Machine Images (AMI's) and the Image Marketplace.
  • Identify and select an existing AMI based on prefereneces such as Linux distribution, installed software packages and libraries.
  • Configure the AMI resources (CPU, memory, storage, network) and deployment region.
  • Learn what is public/private key authentication. Create and upload an SSH key to your instance.
  • Launch and monitor your instance status.
  • Save your launched instance preferences.
  • Access your instance through an SSH client.
  • Terminate your Instance.
Readings and References:
  • "Introduction to Amazon Elastic Compute Cloud (EC2) by Amazon.
  • "Amazon WS Tutorials".
  • GitHub Education Resources.
Week 2: Introduction to Hypertext and Hypermedia
Lecture 3 Introduction to HyperText. Learning outcome:
  • Review the basic principles of Hypertext and its History.
  • Understand the key principles of hypertext (nodes, hyperlinks, navigation) and the properties that hypertext systems need to abide to.
 Readings: Chapters 1-5. Multimedia and Hypertext: The Internet and Beyond. J. Nielsen, AP Professional, 1995.
Lecture 4 Advanced HyperText Concepts. Learning outcome:
  • Overview of Usability. Usability of hypertext systems.
  • Understand the notion of information overloading, network disorientation. Definition of information retrieval and information filtering. Navigation support mechanisms: backtracking, bookmarking, history mechanisms, milestones.
 Readings: Chapters 8-10. Multimedia and Hypertext: The Internet and Beyond. J. Nielsen, AP Professional, 1995.
Lab 1 Introduction to GitHub.
Learning outcome:
  • Learn the basics of Git and GitHub.
  • Learn how to setup and use your Github account and GitHub Education Resources.
Readings and References:
  • "Git from the bottom up" by John Wiegley (2009).
  • "The basics of Git and GitHub" (2013).
  • GitHub Education Resources.
Lab 1 Using GitHub.
Learning outcome:
  • Create a Git repository on GitHub and attach it to EPL425 repository.
  • Familiarize with the Git Dashboard and creating / adding teams.
  • Install SourceTree Git client on your development environment.
  • Configure your development workspace, add your personal webpage HTML source to it.
  • Make the first commit to your Git repository. Make additional changes, commit them and understand the revision tree.
Readings and References:
  • "Git from the bottom up" by John Wiegley (2009).
  • "The basics of Git and GitHub" (2013).
  • GitHub Education Resources.
